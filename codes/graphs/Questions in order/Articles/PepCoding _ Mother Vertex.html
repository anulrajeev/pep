Mother Vertex

Hey there. We are going to discuss an interesting problem under the topic of Graphs, "Mother Vertex".

Before moving forward, we suggest you go through this problem's outline.


Important Links : Problem Link, Solution Video


Problem Discussion
  • A Mother Vertex is a vertex through which we can reach all the other vertices of the Graph.
  • We are given a Directed Graph and have to find a Mother Vertex in the Graph (if present).
  • Reader, there can be multiple mother vertices in a graph as well. We are required to print any one of them.
  • One of the methods for finding the mother vertex is by traversing through every vertex of the graph and checking if we can reach all other vertices through it.
    However, this takes a time complexity of order V*(V+E) which is undesirable. We need to optimize this complexity.
Approach

We consider the graph given in figure 1 to understand this approach.

We apply DFS on the given graph in any order and fill a stack on a N-time basis. N-time basis means adding a node in the stack when backtracking from it.

Let's see how. The visited nodes are marked and can't be visited again.

  • We start the DFS from any node, say node 5. From node 5 we can reach node 6. However, we can't reach any further nodes from node 6, so we add it in our stack and backtrack from the node.
  • After backtracking from node 6, we reach node 5. We realize that we can't traverse anywhere else from this node, so it is added to the stack and we start the DFS from some other node.
  • Now we start the DFS from another node, say node 1. From there we move on to node 2 to node 3 and then to node 4. On reaching node 4, we again realize we can't move any further, so we add node 4 in the stack and backtrack from it.
  • Now we reach node 3. Again, we can't move to any other vertex, so we add node 3 in the stack and backtrack from it, reaching node 2. Similarly, node 2 is added in the stack and we backtrack from it, reaching back to node 1. Since node 1 can't visit any other vertex, it is added in the stack.
  • We still have some vertices left to explore. We now start the DFS process from node 7. We now travel to node 8. Since it's a dead end, we push 8 in the stack and backtrack from it. Now we backtrack to node 7. Since all the other nodes have already been visited, node 7 is also added to the stack.

After the entire process of DFS, we have 2 options: either the vertex on the top of stack i.e. node 7 is the mother vertex or the graph has no mother vertex.

To check if the topmost member of the stack is a mother vertex or not, we again apply DFS on it.

Let us take up one DFS process from node 7 quickly.

From node 7 we reach node 1 to node 2 to node 3 to node 4. Next, we backtrack from node 4, node 3, node 2 and node 1 to reach node 7.

Again, from node 7 we visit node 8 and backtrack back to node 7.

At last, we reach node 5 and then node 6. On backtracking from node 6 and then node 5, we have finally reached node 7 after visiting all the vertices of the graph. So it indeed is a mother vertex.

But, why does this algorithm work? Let's see how from its Mathematical Proof.

MATHEMATICAL PROOF

To understand this, we divide the stack in 2 parts, "ans" and "ros".

Now, four possibilities are there.

  1. There is no mother vertex in either "ans" or "ros".
  2. There is a mother vertex in "ans" but no mother vertex in "ros".
  3. There is a mother vertex in "ans" as well as in "ros".
  4. There is no mother vertex in "ans" but a mother vertex in "ros".

Out of these 4 possibilities, the first 3 possibilities hold ground. We need to analyse the last possibility when the mother vertex is not in the "ans" but in "ros".

For that, let's assume that the vertex 3 is our mother vertex. To validate this, we consider 2 options, either there is a direct/indirect path from vertex 7 to vertex 3 or there is no such path.

  • For the former option, it is true that if we can reach all the vertices from 3 (as it's assumed to be mother vertex) and vertex 7 can reach vertex 3, then vertex 7 can also reach all other vertices and hence vertex 7 should also be a mother vertex.
    This contradicts the possibility that "ans" or vertex 7 is not a mother vertex. Hence, the 4th possibility holds no ground.
  • For the latter option, if there is no direct/indirect path available from vertex 7 to vertex 3, then it is not possible for vertex 7 to appear above vertex 3 is the stack.
    Hence, this is also a contradiction, and again 4th possibility holds no ground.

You can watch the solution video to understand this in detail.

Hence, the 4th possibility is invalid.

Analysis
Time Complexity:

O(V+E)

DFS traversal of the given graph is done and we keep track of the last finished vertex "v" through which takes O(V+E) time.

Also to check if "v" is the mother vertex or not, we again start a DFS from "v" which again takes O(V+E) time.

Hence, the total time complexity is O(V+E)

Space Complexity:

O(N)

Since auxiliary space is used when creating arrays, the space complexity is of order N.
