ALIEN DICTIONARY

Welcome to another problem, dear coder. In this article we are going to discuss a Level-2 Graphs problem, "Alien Dictionary". Doesn't it sound interesting?

Wait, before moving forward, we want you to go through the problem's outline.


Important Links : Problem Link, Solution Video


Problem Discussion
  • You are given a list of strings from the alien language's dictionary, where the strings in words are sorted lexicographically by the rules of this new language.
  • You are required to return a string of the unique letters in the new alien language sorted in lexicographically increasing order by the new language's rules.
  • If there is no solution, return "". If there are multiple solutions, return any of them.

For example, in the English language the array of strings {abc, adc, bore} are in lexicographical (dictionary) order.

The lexicographic increasing order of the unique characters from all the strings according to the English language is {a,b,c,d,e,o,r}.

Similarly, we assume that according to an alien language {wrt, wrf, er, ett, rftt} are lexicographically sorted.

We have to find the lexicographically increasing order of the unique characters from all the strings according to the alien language.

Approach

To begin with the approach, you first need to understand the basic concept of place value used in lexicographic/dictionary order. The characters at a greater place value determine the order of the strings.

For example, in strings "hello" and "iaaaa", the lexicographical order is dependent on the highest place value characters, i.e. 'h' and 'i'. Since 'h' comes before 'i', therefore "hello" comes before "iaaaa" lexicographically despite the use of multiple 'a's in "iaaaa".

If the highest place value characters are equal in both the strings then we compare the second highest place value characters. For example, in strings "abed" and "acrt", since the highest place characters ('a') are the same, therefore we compare 'b' and 'c'.

Now let's discuss how to find the lexicographical order of unique characters for the following strings in an alien language.

Let's first list all the unique characters in the strings given in figure 1.

You must be thinking, one way to find the order of all the characters is to compare every string with every other string.

For example, we compare "wrt" with "wrf" and since "wr" is the same in both, therefore, 't' must come before 'f'. Similarly, we compare "wrt" with "er", "ett", "rftt' and further also compare "wrf" with "wrt", "er", "ett" and so on... This method checks n2 pairs which is not desirable, especially due to redundancies.

The correct method is to compare only the consecutive strings.

Let's see how.

  • First, we compare the strings "wrt" and "wrf". Since "wr" is same in both therefore, we know 't' comes before 'f'.
  • Next, we compare "wrf" with "er", Since the highest place characters are different, therefore, 'w' must come before 'e'.
  • Then "er" and "ett" are compared. According to the second highest place values, 'r' must come before 't'.
  • Finally, by comparing "ett'' with "rftt", we get that 'e' comes before 'r'. Also comparing the second highest place values, we get that 't' comes before 'f' which has already been obtained earlier.

Hence, in figure 6 we obtain a graph which uses all the unique characters in the strings.

Now we simply write the topological order of this graph using the Kahn's algorithm which has been discussed previously. According to it, our answer is {w, e, r, t, f}.

NOTE 1: If we had the input strings as { wrt, wrfz, er, ett, rftt}, then the character 'z' is not compared at any position. Hence, we don't have enough data available to figure out its exact position. So we can print the answer by putting this 'z' at any position in our resultant order.

NOTE 2: We also need to look at an Edge Case. We understand it through an example.

If we have 2 strings { abcd, abc } where "abcd" comes before "abc". When we compare the highest place value characters, "abc" comes to be the same in both the strings but we don't have a mismatched character to compare with 'd'. According to the statement, the string with shorter length must come before. But the given order is opposite to it. Hence, this is not a possible input. In such a case we return an empty string.

import java.io.BufferedReader; import java.io.IOException; import java.io.InputStreamReader; import java.util.*; public class Main { public static void main(String[] args) throws NumberFormatException, IOException { BufferedReader br = new BufferedReader(new InputStreamReader(System.in)); int n = Integer.parseInt(br.readLine()); String[] words = br.readLine().split(" "); System.out.println(alienOrder(words)); } public static String alienOrder(String[] words) { HashMap< Character,HashSet< Character>>map=new HashMap< >(); HashMap< Character,Integer>indegree=new HashMap< >(); for(String st: words){ for(char ch:st.toCharArray()){ indegree.put(ch,0); } } for(int i=0;i< words.length-1;i++){ String curr=words[i]; String next=words[i+1]; boolean flag=false; int len=Math.min(curr.length(),next.length()); for(int j=0;j< len;j++){ char ch1=curr.charAt(j); char ch2=next.charAt(j); if(ch1!=ch2){ HashSet< Character>set=new HashSet< >(); if(map.containsKey(ch1)==true){ set=map.get(ch1); if(set.contains(ch2)==false){ set.add(ch2); indegree.put(ch2,indegree.get(ch2)+1); map.put(ch1,set); } }else{ set.add(ch2); indegree.put(ch2,indegree.get(ch2)+1); map.put(ch1,set); } flag=true; break; } } if(flag==false && curr.length()>next.length()){ return ""; } } LinkedList< Character> queue=new LinkedList< >(); StringBuilder ans=new StringBuilder(); for(char ch:indegree.keySet()){ if(indegree.get(ch)==0){ queue.addLast(ch); } } int count=0; while(queue.size()>0){ char rem=queue.removeFirst(); ans.append(rem); count++; if(map.containsKey(rem)==true){ HashSet< Character>nbrs=map.get(rem); for(char nbr:nbrs){ indegree.put(nbr,indegree.get(nbr)-1); if(indegree.get(nbr)==0){ queue.addLast(nbr); } } } } if(count==indegree.size()){ return ans.toString(); }else{ return ""; } } }
